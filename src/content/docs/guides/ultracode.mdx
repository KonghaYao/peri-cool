## 概述

ultracode 是 Peri 的工作流引擎。你把一个复杂任务描述给 Peri，它自动拆分成多个子代理并行或分阶段执行。

使用方式：`/ultracode <任务描述>`。

## 使用场景

### 重构

```
/ultracode 重构 src/services/order.ts，每个函数先检查类型错误，再改为 async/await，最后审查逻辑正确性。完成后输出变更摘要。
```

### 代码审查

```
/ultracode 审查这次改动的 5 个文件，从安全性、性能、代码风格三个角度并行审查。输出问题清单。
```

### 批量迁移

```
/ultracode 把 src/components/ 下 8 个 class 组件改成函数组件 + Hooks。每个组件先分析 props/state，再重写，再检查类型。
```

### 多维度分析

```
/ultracode 分析 src/api/ 模块，从调用链、错误处理、接口设计三个维度同时分析。汇总问题列表。
```

## 执行模式

大多数时候不需要管模式，Peri 自己判断。以下只是在自动拆分结果不对时用来限定行为：

| 模式 | 适用情况 | 关键词 |
|------|---------|--------|
| **并行** | 子任务互不依赖 | 同时、并行、分别 |
| **阶段化** | 有步骤依赖，串行跑 | 先...再...然后...、分阶段 |
| **管线** | 每个文件走同样的多步流程 | 每个...都先...再... |

## 不适合的场景

- 子任务有**紧耦合依赖**（改 B 之前必须知道 A 的改动结果）
- 子任务**共享同一批文件**（并行写同一个文件会冲突）
- 拆分开销大于执行开销（改 2 个文件没必要并行）

## 并发限制

默认最多 3 个并行子代理。免费版 DeepSeek 建议 2 个。

可以在命令中指定：**"最多 2 个并行"**。

## 结果

执行完成后，对话中直接展示汇总结果。完整日志保存在 `.claude/workflow-runs/` 下。

## 什么是 ultracode

ultracode 是 Peri 的工作流（Workflow）引擎。它和子代理（SubAgent）的关系是——子代理一次派发几个 agent 并行做一件事，ultracode 一次派发几十个 agent 跑一个结构化流程。

区别在于谁掌握编排逻辑：

- **子代理**：Peri（LLM）逐轮判断下一步该做什么
- **ultracode**：你（或 Peri 生成的）JavaScript 脚本定义整个流程

## 为什么需要 ultracode

<CardGrid>
  <Card title="规模" icon="layers">
    一次审查 5 个文件用子代理。扫描整个代码库 200 个文件找 dead code？用 ultracode。
  </Card>
  <Card title="可重复" icon="refresh">
    工作流脚本是代码。审查流程可以保存、重跑、分享。子代理的决策在 LLM 的上下文里，无法重放。
  </Card>
  <Card title="质量控制" icon="check">
    脚本可以定义交叉审查——两个 agent 独立检查同一个问题，不一致时启动第三个裁决。
  </Card>
  <Card title="不阻塞" icon="play">
    工作流在独立 Node.js 进程中运行。你继续在主会话工作，完成后收到通知。
  </Card>
</CardGrid>

## ultracode vs 子代理 对比

| | 子代理 | ultracode 工作流 |
|---|--------|-----------------|
| 谁决定下一步 | LLM 逐轮判断 | 脚本定义 |
| 并行规模 | 每轮 1-3 个 | 几十到上百个 |
| 可重复性 | 不可重放 | 脚本可重跑 |
| 中间结果 | 在 LLM 上下文中 | 在脚本变量中 |
| 适用场景 | 单次委派、快速搜索 | 批量审计、大规模迁移 |

## 架构速览

```text
用户输入 /ultracode 任务描述
       ↓
Peri (LLM) 生成 JavaScript ESM 脚本
       ↓
Node.js 进程执行脚本（npx / bunx）
       ↓
JSON-RPC 双向通信
       ↓
peri-acp 调度 sub-agents 执行
       ↓
结果汇总 → 通知用户
```

## 使用方式

`/ultracode &lt;任务描述&gt;`

不需要写脚本。Peri 会根据你的任务描述自动生成工作流脚本并执行。

<Aside type="tip">
  需要手动控制时，可以编写脚本放到 `.claude/workflows/` 目录，然后通过 `/workflows` 面板选择执行。
</Aside>

---
title: 快速上手新项目
description: 刚 clone 一个代码库，让 Peri 帮忙快速理解架构和模块职责
---

刚接手陌生代码库，最差的策略是打开文件逐个读。你需要的是宏观认知，不是细节淹没。

## 第一步：读 CLAUDE.md

Peri 进入项目自动加载 CLAUDE.md，里面记录了项目规则、技术栈、约定俗成。不需要额外说明——如果项目没有，先创建一个。这是后续所有对话的"上下文地基"。

## 第二步：让 Peri 总结架构

不需要花哨的 skill，直接说：

```text
先读 CLAUDE.md 了解项目规则，Glob 项目结构，然后读关键入口文件，
总结：1. 项目架构和模块职责 2. 数据流向 3. 关键设计决策
```

Peri 会并行扫描目录结构、入口文件、配置文件，十分钟内给你一份全局视图。

## 第三步：派 explorer 子代理深挖特定模块

看完架构总结，对感兴趣的模块不要自己一个个文件读——派子代理做独立探索，避免撑满主代理上下文：

```text
派 explorer 子代理理解 src/auth/ 的认证流程，给我结构化总结：
1. 从 HTTP 请求进来到返回 token，经过哪些模块
2. 每个模块的职责边界
3. 错误处理链路
```

子代理独立探索、独立汇总，主代理上下文干净如初，你拿到的是一份精炼报告。

## 第四步：问答式挖掘

摸清整体结构后，像问同事一样问 Peri：

```text
数据库操作统一入口在哪？用了什么抽象层？
哪些模块测试覆盖率最低？
错误处理有没有统一规范？
```

Peri 自己会搜、会统计——你不需要知道在哪查，只需要提出正确的问题。

## 省钱提醒

探索阶段只用便宜模型——读代码、搜调用链、总结模块，这些消耗 token 但不要求推理深度：

```text
/model deepseek-chat
// 50-80 条消息扫完整个项目，费用几乎为零
```

摸清楚了再切 Sonnet 做实际改动。这是合理用法，不是乱省钱。

## 架构审计（进阶）

当你已经对项目有基本了解、想做深度改进时，用 `improve-codebase-architecture`：

```text
/improve-codebase-architecture src/ 范围
```

> 安装：`npx skills add https://github.com/konghayao/peri --skill improve-codebase-architecture`

它会扫描模块职责、依赖关系、设计模式，输出结构化的架构改进建议。注意：这是**审计工具**，适合已熟悉的项目，不是陌生项目的入门手段。

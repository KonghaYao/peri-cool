Peri 的 `Agent` 工具有三种执行模式，通过两个参数控制：

- `fork`（默认 false）：是否继承父会话的完整上下文
- `run_in_background`（默认 false）：是否后台运行

三种模式按组合划分：默认（同步模式）、`fork: true`（Fork 模式）、`run_in_background: true`（后台模式）。

## 三种模式对比

| 特性 | 同步模式 | Fork 模式 | 后台模式 |
|------|---------|----------|---------|
| 上下文继承 | 独立 | 完整继承父会话 | 独立 |
| 主会话阻塞 | 是 | 是 | 否 |
| 结果获取 | 立即可用 | 立即可用 | 异步通知 |
| 适用场景 | 单次委派 | 连续协作 | 独立并行 |

## 同步模式

默认模式。主 Agent 调用子代理后等待结果返回，拿到输出后继续推理。

适合委派一个独立的子任务——搜索代码库的某个模式、审查一个文件、查一段文档。子代理有自己的上下文窗口，不会干扰主会话的对话历史。

```json
// Agent("explorer", "在 src/ 中查找所有 TODO 注释")
{ "subagent_type": "explorer", "prompt": "在 src/ 中查找所有 TODO 注释" }
```

## Fork 模式

Fork 模式下子代理继承父会话的完整上下文——所有对话历史、系统提示词和工具集。子代理能"看到"主会话从开始到当前的全部消息。

适合需要父会话背景的连续协作。例如：先在主会话中探索代码结构，然后 Fork 一个 coder 基于已有发现开始编码。Fork 子代理不需要你重新解释背景，它在已有对话上下文中继续工作。

```json
// Agent("继续上一个任务，把用户服务从回调风格改成 async/await")
{ "fork": true, "prompt": "继续上一个任务，把用户服务从回调风格改成 async/await" }
```

<Aside type="note">
  Fork 模式复制的是调用时刻的上下文快照。Fork 子代理启动后，父会话的新消息不会同步到它。
</Aside>

Fork 子代理的输出遵循固定格式——Scope（范围）、Result（结果）、Key files（关键文件）、Files changed（修改的文件）——便于主 Agent 解析和继续使用。

## 后台模式

Fire-and-forget。主 Agent 启动子代理后立即继续推理，不等待结果。子代理完成后，通过系统通知把结果推送到主会话的消息流中。

适合可独立进行的任务——批量文件搜索、长运行测试、代码库审计。主会话不阻塞，你可以在等待时继续和 Peri 对话。

```json
// Agent("web-researcher", "搜索 Tokio 最新版本的功能变更", background)
{ "subagent_type": "web-researcher", "prompt": "搜索 Tokio 最新版本的功能变更", "run_in_background": true }
```

<Aside type="tip">
  后台模式最多 3 个并行。可以在任务描述中指定需要的数量。
</Aside>

## 模式选型

选择执行模式时按以下顺序判断：

1. **需要父会话上下文才能正确执行**，比如"基于我们刚才的讨论继续做 X" → Fork 模式
2. **任务独立，不想等**，比如搜索、审查、测试 → 后台模式
3. **任务独立，可以等** → 同步模式（默认，也是大多数场景的选择）

子代理是 Peri 在后台并行启动的独立 Agent。每个子代理有自己的上下文窗口和工具集，可以同时做不同的事——一个在写代码，一个在审查，一个在搜索文档。

## 使用方式

你不需要手动启动子代理。Peri 根据任务自动判断是否需要并行，然后通过 `Agent` 工具派发：

```
你: "帮我重构 src/services/，先分析结构，再并行重构每个文件，最后汇总变更"
```

Peri 会拆成多个子代理，并行跑分析、重构、审查。

<Aside type="note">
  子代理有专门的类型——coder、explorer、code-reviewer、web-researcher 等——每种类型有不同的工具权限和行为模式。
</Aside>

## 代理类型

| 类型 | 用途 | 工具权限 |
|------|------|---------|
| **coder** | 写代码、改文件 | 全部文件操作 |
| **explorer** | 搜索代码库 | 只读 |
| **code-reviewer** | 审查变更 | 只读 |
| **web-researcher** | 搜索网页、抓取文档 | 网络工具 |
| **plan** | 设计实现方案 | 只读 |

## 后台执行

默认最多 3 个并行子代理。你可以在任务描述中指定数量。

子代理运行期间，主会话不受影响——你可以在聊天区继续提问，Peri 会在子代理完成后通知你。

<Aside type="tip">
  对于免费版 API（如 DeepSeek），建议限制为 2 个并行，避免触发速率限制。
</Aside>

## 适用场景

- 多文件并行重构
- 代码审查 + 修复同步进行
- 搜索文档 + 应用搜索结果的组合任务
- 探索代码库结构的同时进行编码

## 什么是子代理

子代理（SubAgent）是 Peri 在后台并行启动的独立 Agent 实例。每个子代理拥有独立的上下文窗口（Context Window）和工具集——一个在写代码时，另一个可以同时审查，第三个在搜索文档。子代理不阻塞主会话，你可以在等待结果时继续与 Peri 对话。

与手动切换任务不同，Peri 会根据任务复杂度自动判断是否需要并行，然后通过 `Agent` 工具派发子代理。你只需要描述任务目标，Peri 负责选择合适的子代理类型、准备输入、在后台启动并收集结果。

子代理在单个会话内运行。如需跨多个独立会话的并行任务，请查阅 [ultracode 工作流](/docs/features/ultracode/)。

## 子代理如何工作

每个子代理在自己的独立上下文中运行，拥有一套完整的系统提示词、工具集和权限配置。子代理会复用主 Agent 在会话开始时捕获的冻结数据（CLAUDE.md、Skills 摘要等），确保看到的环境与主 Agent 一致。

主 Agent 根据任务描述选择合适的子代理类型，为其准备输入参数，然后在后台启动。启动后，主 Agent 继续执行当前任务，不被阻塞。子代理完成后，结果通过后台任务通知推送到主会话的消息队列中，主 Agent 在下一轮会收到这些结果。

<Aside type="note">
  子代理默认最多 3 个并行执行。你可以在任务描述中指定需要的并行数量。对于速率限制较严格的 API（如免费版 DeepSeek），建议减少到 2 个并行以避免触发限制。
</Aside>

## 内置子代理

Peri 内置了 7 种专业子代理，每种有不同的工具权限和行为模式。Peri 根据任务需求自动选择合适的类型，你也可以在描述中显式指定。

| 类型 | 用途 | 工具权限 | 模型层级 |
|------|------|---------|---------|
| **explorer** | 搜索代码库、发现文件模式 | 只读 | haiku（快速低成本） |
| **plan** | 设计实现方案、分析架构 | 只读 | 继承主会话 |
| **coder** | 写代码、改文件、重构 | 全部文件操作 | sonnet |
| **code-reviewer** | 审查代码质量、安全、可维护性 | 只读 | sonnet |
| **web-researcher** | 网页搜索、文档抓取、信息收集 | 网络工具 | haiku |
| **verification** | 验证实现正确性，运行构建和测试 | 只读 | sonnet |
| **general-purpose** | 通用复杂研究、多步骤任务 | 继承主会话 | 继承主会话 |

每种类型的详细权限、策略和适用场景见 [内置子代理详解](/docs/features/subagent-builtin/)。

## 三种执行模式

Peri 提供三种子代理执行模式，每种适用不同的协作场景：

- **同步模式（sync）**：主 Agent 等待子代理完成后继续。适用于子代理结果直接影响后续决策的场景。
- **Fork 模式**：子代理继承主会话的完整对话历史，在独立上下文中继续工作。适用于需要完整上下文感知的分支任务。
- **后台模式（background）**：fire-and-forget，主 Agent 不等待。结果通过通知推送回来，适用于独立无依赖的批处理任务。

模式选择和参数控制详见 [执行模式详解](/docs/features/subagent-modes/)。

## 自定义子代理

除了内置子代理，你还可以创建自定义子代理来处理特定领域任务。在 `.claude/agents/` 目录下放置 Markdown 文件即可定义一个子代理——文件名即类型名，YAML frontmatter 声明模型、工具权限、系统提示词。

自定义子代理可以设定专属的系统提示词，让子代理聚焦某个特定领域——比如"只审查安全漏洞"或"只处理数据库迁移"。也可以限制工具集，防止子代理执行超出预期的操作。

创建方式和完整配置项详见 [自定义子代理](/docs/features/subagent-custom/)。
